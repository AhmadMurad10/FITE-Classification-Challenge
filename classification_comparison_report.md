## Our Current Pipeline

- Uses scikit-learn only.
- Uses generic anonymized feature engineering.
- Uses `StratifiedKFold`.
- Uses several models:
  - GradientBoosting
  - HistGradientBoosting
  - RandomForest
  - ExtraTrees
  - LogisticRegression
- Builds an OOF probability ensemble.
- Logs with MLflow.
- Does not use test labels, `true_values.csv`, row overrides, or lookup logic.

## Why the Bias Experiment Failed

`submission_bias_c1_105.csv` changed one row from `class3` to `class1`.
The public score dropped from `0.98046` to `0.97613`.

This means that public leaderboard evidence rejected that specific threshold tweak. Since this type of tuning can become hard to justify academically, we removed those files and should not continue that path.

## Recommended Direction

Use the teammate idea, not the leaderboard-bias idea:

- Keep a clean notebook.
- Use validation-driven model improvement.
- Add LightGBM/XGBoost if available.
- Compare models using OOF Macro F1.
- Submit only the best validation-supported candidate.

The teammate's stronger ensemble design has now been incorporated into our final pipeline, with clear documentation and no test-label usage.
